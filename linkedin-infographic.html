01
Strict Directory Contract
app/           → Pages (file-based routing)
components/     → Reusable UI (PascalCase)
services/       → All API calls (single entry)
hooks/          → Business logic only
store/          → Zustand (client state)
constants/      → Design tokens
types/          → TypeScript definitions
02
State Boundaries
No overlap. No confusion about source of truth.
Zustand
Auth · Onboarding · UI State
React Query
All Server Data · Cache · Sync
Rule: Server data never lives in Zustand. Ever.
03
One API Layer — Zero Component Changes at Launch
Every API call flows through services/api.ts. During dev, hooks return mock data. When backend goes live, we swap the service layer — not a single component changes.
