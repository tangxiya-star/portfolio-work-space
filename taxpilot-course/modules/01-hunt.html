What the App Asks You — and Why

Onboarding isn't a survey — it's a tax-rule selector. Every answer you give maps to a specific IRS category that changes what counts as deductible.

CODE 
const INCOME_OPTIONS = [
  { key: '1099',   label: 'Self-employed income (1099)' },
  { key: 'side',   label: 'Side income' },
  { key: 'W2',    label: 'W-2 Salary' },
  { key: 'K1',    label: 'K-1 Distribution' },
  { key: 'rental', label: 'Rental income' },
]

const ENTITY_OPTIONS = [
  { key: 'LLC',         label: 'LLC (Single Member)' },
  { key: 'individual',  label: 'Individual (No Entity)' },
  { key: 'S-Corp',      label: 'S-Corp' },
  { key: 'partnership', label: 'Multi-member LLC / Partnership' },
]
PLAIN ENGLISH

These short keys map to IRS tax category codes. The app stores 1099, not the full label — so the backend knows exactly which rules apply.

1099 = paid without tax withheld. Freelancers and contractors get 1099 forms. They can deduct home offices, equipment, and software that W-2 employees cannot.

LLC vs S-Corp = different profit rules. An LLC owner's profits flow straight to personal taxes. An S-Corp can pay the owner a salary and split the rest — changing which deductions apply.

Your answers determine your deduction rulebook. Same coffee shop expense = deductible for a 1099 freelancer, not deductible for a W-2 employee — TaxPilot needs to know which you are.

💡
Aha! The onboarding questions aren't random — they determine which IRS deduction rules apply to you. Change the answer, change the rules.

Discovery — Your First Report

After Plaid delivers the transaction history, the Discovery screen shows one of four possible states. Each tells a different story about the user's account.

✅

Case 1 — All Clear

Every deductible expense was auto-confirmed. The user sees their total savings with no action required.

status: confirmed
🔶

Case 2 — Partial

Some transactions confirmed automatically, others flagged for manual review. User taps through the ambiguous ones.

status: mixed
❓

Case 3 — All Pending

Every transaction needs the user's confirmation. Common when expense categories are broad or overlap.

status: pending
🆕

Case 4 — No History

No prior transactions found in the connected account. Tracking starts from today — the slate is clean.

status: no_history
