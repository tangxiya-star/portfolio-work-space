The Design Engine

TaxPilot's entire visual identity — every green, every font, every rounded corner — flows from a single 196-line file called theme.ts.

Think of it like a fashion house style guide. One master document defines all approved colours, fabrics (fonts), and silhouettes (spacing). Any deviation gets flagged immediately. The system is built in three tiers.

The Colour Pyramid

TIER 1 — BRAND PRIMITIVES

ink
newsprint
emerald
white

4 raw hex values. Never used directly in components.

TIER 2 — SEMANTIC TOKENS

textPrimary → ink
background → newsprint
accent → emerald
border → ink @ 10%

Meaningful names that reference primitives. Screens use these.

TIER 3 — COMPONENT USAGE

HomeScreen
TransactionCard
SavingsScreen
CalendarScreen
PendingPill
15+ more…

Screens use semantic names only — never raw hex values.

CODE — constants/theme.ts (colour section) 

const brand = {
  ink: '#0D0D0D',
  newsprint: '#E3DFD5',
  emerald: '#22C55F',
  white: '#FFFFFF',
} as const

export const theme = {
  colors: {
    ...brand,
    primary: brand.ink,
    accent: brand.emerald,
    background: brand.newsprint,
    surface: brand.white,
    textPrimary: brand.ink,
    textSecondary: hexToRgba(brand.ink, 0.6),
    textTertiary: hexToRgba(brand.ink, 0.5),
    border: hexToRgba(brand.ink, 0.1),
    green: brand.emerald,
    greenBg: hexToRgba(brand.emerald, 0.1),
    amber: '#C2410C',
    amberBg: 'rgba(194, 65, 12, 0.1)',
  },
PLAIN ENGLISH

brand — the 4 raw colour values. The as const tells TypeScript these strings are exact literals, not just any string. These are never referenced directly in components.

theme.colors builds semantic names on top of the primitives. textPrimary means ink. accent means emerald. This indirection is the entire point of the system.

hexToRgba() creates semi-transparent versions. textSecondary is ink at 60% opacity — a softer label colour without introducing a new hex value.

amber is the one exception — a status colour with no primitive alias. It still lives in theme.ts so it has a single source of truth.

⚠️
CLAUDE.md rule: Never hardcode hex values in components. If AI writes color: '#0D0D0D' in a screen, flag it immediately — it should be theme.colors.ink.

MODULE 4 / SCREEN 2

Typography — Font Styles as Variables

Instead of remembering font sizes and weights, components import pre-built style objects. One name, all the right values, guaranteed consistent.

CODE — constants/typography.ts (display + title styles) 

export const typography = {
  display: {
    fontFamily: theme.fonts.display,
    fontSize: theme.fontSize.display,
    lineHeight: theme.lineHeight.display,
    fontWeight: '700',
    letterSpacing: theme.letterSpacing.tight,
    color: theme.colors.textPrimary,
  } as TextStyle,

  title1: {
    fontFamily: theme.fonts.display,
    fontSize: theme.fontSize.title1,
    lineHeight: theme.lineHeight.title1,
    fontWeight: '700',
    letterSpacing: theme.letterSpacing.snug,
    color: theme.colors.textPrimary,
  } as TextStyle,

  title2: {
    fontFamily: theme.fonts.heading,
    fontSize: theme.fontSize.title2,
    lineHeight: theme.lineHeight.title2,
    fontWeight: '600',
    letterSpacing: theme.letterSpacing.snug,
    color: theme.colors.textPrimary,
  } as TextStyle,
PLAIN ENGLISH

display — 34px Space Grotesk Bold with tight letter-spacing. This is the style used for the dollar amount hero on the home screen. Large, heavy, commanding.

title1 — 28px, also Space Grotesk Bold. Used for section headers like "Your Deductions" or year labels. Same weight as display, smaller scale.

title2 — 22px, switching to the heading font at weight 600. Slightly lighter — used for card titles and subsection labels.

Each style is a pre-built object. A component just writes style={'{'}typography.display{'}'} and gets every property at once — no guessing, no drift.

StyleSheet.create — Styles as Objects

React Native has no CSS files. Instead, styles are JavaScript objects — but the pattern maps directly to CSS concepts you already know.

CODE — PendingPill StyleSheet block 

const pendingStyles = StyleSheet.create({
  pill: {
    flexDirection: 'row',
    alignItems: 'center',
    alignSelf: 'flex-start',
    backgroundColor: theme.colors.surface,
    paddingHorizontal: theme.spacing.lg,
    paddingVertical: theme.spacing.sm,
    minHeight: 44,
    borderRadius: theme.radius.round,
    gap: theme.spacing.sm,
  },
  pillPressed: {
    backgroundColor: theme.colors.ink,
  },
  text: {
    fontWeight: '600',
    color: theme.colors.textPrimary,
  },
  textPressed: {
    color: theme.colors.white,
  },
})
PLAIN ENGLISH

StyleSheet.create() is like writing CSS classes. Each key (pill, text) is a named style object you apply to a component — exactly like a CSS class name.

flexDirection: row = CSS display: flex; flex-direction: row. paddingHorizontal = CSS padding-left + padding-right combined into one shorthand.

minHeight: 44 is an Apple HIG requirement. Touch targets must be at least 44×44 points. This is an accessibility rule, not a preference.

pillPressed / textPressed are the pressed-state overrides. The component applies both style arrays simultaneously when the button is active: style={'{'}[styles.pill, isPressed && styles.pillPressed]{'}'}.

👆
minHeight: 44 is not optional. Apple Human Interface Guidelines require touch targets to be at least 44×44 points. Too small means users miss taps — especially for people with motor impairments. This is an accessibility requirement baked into every interactive component in TaxPilot.

One File to Rule Them All

A single change to theme.ts ripples through the entire app instantly. No find-and-replace. No missed screens. No broken consistency.

🎨

SCENARIO 1

Change brand.emerald to orange #FF6B35

Everything referencing accent, green, or greenBg updates automatically:

Hero card background Confirmed badges Year indicator Savings progress bar Calendar highlights + 15 more screens
📐

SCENARIO 2

Change theme.spacing.lg from 16 to 24

Every component using theme.spacing.lg respaces simultaneously:

Button padding Card padding Nav item spacing Input padding PendingPill horizontal
✨

SCENARIO 3

Add a new colour coral to the palette

Define it once in theme.ts. It becomes available everywhere, immediately, with full TypeScript autocomplete:

coral: '#FF6B6B',   // add this one line
theme.colors.coral  // now valid everywhere
